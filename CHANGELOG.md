# Changelog

Все значимые изменения в этом проекте будут документированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и этот проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [1.2.4] - 2026-03-27

### Изменено

- Не проверять наличие уникального индекса на поле `title` в коллекции `events`
в рамках лабораторной работы №3, так как это не даст создать несколько событий с одинаковым названием,
что необходимо для выполнения других лабораторных работ

### Исправлено

- Падение АТ при проверке лабораторной работы №3 при выполнении лабораторной работы №4,
которая требует возможности создавать несколько событий с одинаковым названием

## [1.2.3] - 2026-03-18

### Изменено

- Проверка сессии в куках при неудачной регистрации пользователя
- Обновлено ТЗ лабораторной работы №3 в части описания поведения при регистрации пользователя

### Исправлено

- Исправлена некорректная проверка наличия сессии в куках при неудачной регистрации пользователя,
которая не учитывала возможность её отсутствия,
что приводило к ошибкам в АТ при проверке ответа на регистрацию с некорректными данными

## [1.2.2] - 2026-03-16

### Изменено

- Наименования эндпоинтов в ТЗ лабораторной работы №3:
  - `POST /user` → `POST /users`
  - `POST /event` → `POST /events`
- АТ для проверки статуса ответа при неудачной аутентификации

### Исправлено

- Исправлены несоответствия REST API в ТЗ лабораторной работы №3
- Исправлен АТ для проверки статуса ответа при неудачной аутентификации, который ожидал статус `400 Bad Request`
вместо `401 Unauthorized`, что не соответствовало должному поведению тестируемого приложения
- Опечатка в ответе успешного создания события в ТЗ лабораторной работы №3: `201 OK` → `201 Created`

## [1.2.1] - 2026-03-13

### Добавлено

- Отдельные конфиги для лабораторных работ в `config/lab2` и `config/lab3`

### Изменено

- При запуске каждой лабораторной работы теперь используется соответствующий конфиг вместо общего

### Исправлено

- Исправлена валидация конфига в лабораторной работе №2, которая некорректно проверяла наличие полей,
связанных с MongoDB, что приводило к ошибкам при запуске воркфлоу АТ в репозиториях студентов,
не использующих MongoDB для выполнения этой лабораторной работы

## [1.2.0] - 2026-03-13

### Добавлено

- **Лабораторная работа №3 :**
  - Техническое задание с описанием работы с сессиями
  - АТ для проверки выполнения лабораторной работы
  - Джоба `lab3` в воркфлоу EventHub
  - Переиспользуемый воркфлоу в EventHub для проверки лабораторной работы `.github/workflows/reusable_eventhub_lab3.yml`,

- **Новые пакеты АТ:**
  - `client/mongo` — клиенты для HTTP и Mongo
  - `config/mongo` — конфигурация приложений и подключений к MongoDB
  - `test/integration/client/mongo` - интеграционные тесты для проверки взаимодействия с MongoDB
  - `test/fake/client/mongo` — фейковые реализации клиента MongoDB для использования в тестах
  - `app/cookie/session` - для работы с куки в рамках АТ
  - `app/mongo` - для маппинга объектов Mongo к доменным
  - `app/redis` - для маппинга объектов Redis к доменным
  - `app/user` - доменные структуры пользователя тестируемого приложения
  - `app/event` - доменные структуры событий тестируемого приложения
  - `app/endpoint` — струтуры запросов и ответов для новых эндпонитов в рамках лаборатораторной работы №3
  - `expect/http/response`: assert-функции для проверки HTTP-ответов

- **Расширения существующих пакетов:**
  - `expect/strings`: новые assert-функции для проверки строк
  - `expect/times`: новые assert-функции для проверки временных значений
  - `app/endpoint` — получение новых эндпоинтов сервисов их параметры запросов, для которых выполняются АТ

- **Go зависимости:**
  - `go.mongodb.org/mongo-driver/v2` для взаимодействия с MongoDB
  - `github.com/sitnikovik/fluxhttp` для мержа нескольких наборов query-параметров

## [1.1.0] - 2026-02-28

### Добавлено

- **Лабораторная работа №2 :**
  - Техническое задание с описанием работы с сессиями
  - АТ для проверки выполнения лабораторной работы
  - Джоба `lab2` в воркфлоу EventHub
  - Переиспользуемый воркфлоу в EventHub для проверки лабораторной работы `.github/workflows/reusable_eventhub_lab2.yml`,

- **Инфраструктура тестирования:**
  - Воркфлоу `.github/workflows/tests.yml` для запуска модульных и интеграционных тестов
  - `docker-compose.test.yml` с Redis для интеграционных тестов
  - `.github/ci.env` с минимальным порогом покрытия кода (70%)
  - Команды Makefile: `unit-test`, `integration-test`, `coverage`, `coverage-check`
  - Скрипты покрытия кода в директории `scripts/coverage/`

- **Новые пакеты АТ:**
  - `autograder` — ядро системы автоматической проверки с поддержкой шагов выполнения
  - `client/httpx` и `client/redis` — клиенты для HTTP и Redis
  - `config` — конфигурация приложений и подключений
  - `step` — управление шагами АТ с поддержкой переменных, передаваемых между шагами
  - `test/integration` - интеграционные тесты для кода АТ
  - `test/fake` — фейковые реализации зависимостей для использования в тестах
  - `timex` — пакет для работы со временем
  - `user/session` — работа с пользовательскими сессиями
  - `app/endpoint` — получение эндпоинтов сервисов, для которых выполняются АТ

- **Расширения существующих пакетов:**
  - `paints/log`: функции `Duration()` и `Time()` для форматированного вывода
  - `expect/numbers`: новые assert-функции для проверки чисел
  - `expect/strings`: новые assert-функции для проверки строк
  - `expect/times`: новые assert-функции для проверки временных значений
  - `errs`: вложенная обработка ошибок с поддержкой стека вызовов и дополнительной информации

- **Go зависимости:**
  - `github.com/redis/go-redis/v9` для взаимодействия с Redis
  - `github.com/stretchr/testify` для тестирования

### Изменено

- **GitHub Actions:**
  - Джоба `test` перенесена из воркфлоу "Quality" в отдельный воркфлоу "Tests"
  - Джоба линтера переименована: `lint` → `linter`

- **Makefile:**
  - Команда `check` теперь включает проверку покрытия кода (`coverage-check`)
  - Команда `test` разделена на `unit-test` и `integration-test` с раздельными coverage-профилями

### Удалено

- Линтер `paralleltest` из конфигурации `golangci-lint` для обеспечения стабильности интеграционных тестов
- Файл `autograder/internal/expect/error.go` (функциональность перенесена в другие пакеты)

### Исправлено

- Git hooks (`.githooks/pre-push`, `.githooks/pre-commit`): добавлен флаг `set -e` для корректной остановки выполнения при ошибках

## [1.0.0] - 2026-02-06

Первый публичный релиз системы автоматической проверки лабораторных работ по NoSQL.

### Добавлено

- Общая документация курса
- Документация лабораторной работы №1 "Healthcheck"
- GitHub Actions workflows:
  - **EventHub** для автоматической проверки лабораторных работ
  - **Quality** для проверки качества кода (тесты + линтер)
  - **Release** для автоматической публикации релизов
- АТ для лабораторной работы №1
- Вспомогательные пакеты АТа:
  - *console*: логирование, обработка паник и фатальных ошибок
  - *env*: работа с переменными окружения
  - *errs*: расширенная обработка ошибок
  - *expect*: проверка ожиданий в тестах
  - *json*: парсинг JSON body
  - *paints*: раскраска вывода в логи
- Makefile с командами для разработки
  - *test*: запуск тестов
  - *lint*: запуск линтера
  - *check*: запуск всех проверок перед пушем ветки
  - *githooks*: установка Git hooks для автоматической проверки кода
- Git hooks
  - *pre-commit*: запуск тестов перед коммитом
  - *pre-push*: запуск всех проверок качества проекта перед пушем ветки
- Конфигурация **golangci-lint**
  - *govet*: проверка кода на наличие потенциальных ошибок
  - *staticcheck*: статический анализ кода для выявления ошибок и улучшения качества
  - *unused*: выявление неиспользуемого кода
  - *errcheck*: проверка обработки ошибок
  - *revive*: помогает поддерживать чистый и читаемый код
  - *misspell*: проверка орфографии в коде и комментариях
  - *unparam*: выявление неиспользуемых параметров в функциях
  - *gocyclo*: измерение цикломатической сложности функций и методов
  - *paralleltest*: обязательное использование параллельного выполнения тестов (если это возможно)
  - *godoclint*: проверка комментариев на соответствие стандарту GoDoc
  - *godot*: проверка наличия точек в конце комментариев
  - *godox*: выявление TODO, FIXME и других подобных комментариев в коде
  - *dogsled*: выявление чрезмернного количества игнорируемых возвращаемых значений
  - *bodyclose*: проверка закрытия тела ответа HTTP
  - *testpackage*: проверка, что тесты находятся в отдельном пакете от реализации
