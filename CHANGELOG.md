Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),

## [1.0.0] - 2026-02-06

Первый публичный релиз системы автоматической проверки лабораторных работ по NoSQL.

### Добавлено

- Общая документация курса
- Документация лабораторной работы №1 "Healthcheck"
- GitHub Actions workflows:
  - **EventHub** для автоматической проверки лабораторных работ
  - **Quality** для проверки качества кода (тесты + линтер)
  - **Release** для автоматической публикации релизов
- Автогрейдер для лабораторной работы №1
- Вспомогательные пакеты автогрейдера:
  - *console*: логирование, обработка паник и фатальных ошибок
  - *env*: работа с переменными окружения
  - *errs*: расширенная обработка ошибок
  - *expect*: проверка ожиданий в тестах
  - *json*: парсинг JSON body
  - *paints*: раскраска вывода в логи
- Makefile с командами для разработки
  - *test*: запуск тестов
  - *lint*: запуск линтера
  - *check*: запуск всех проверок перед пушем ветки
  - *githooks*: установка Git hooks для автоматической проверки кода
- Git hooks
  - *pre-commit*: запуск тестов перед коммитом
  - *pre-push*: запуск всех проверок качества проекта перед пушем ветки
- Конфигурация **golangci-lint**
  - *govet*: проверка кода на наличие потенциальных ошибок
  - *staticcheck*: статический анализ кода для выявления ошибок и улучшения качества
  - *unused*: выявление неиспользуемого кода
  - *errcheck*: проверка обработки ошибок
  - *revive*: помогает поддерживать чистый и читаемый код
  - *misspell*: проверка орфографии в коде и комментариях
  - *unparam*: выявление неиспользуемых параметров в функциях
  - *gocyclo*: измерение цикломатической сложности функций и методов
  - *paralleltest*: обязательное использование параллельного выполнения тестов (если это возможно)
  - *godoclint*: проверка комментариев на соответствие стандарту GoDoc
  - *godot*: проверка наличия точек в конце комментариев
  - *godox*: выявление TODO, FIXME и других подобных комментариев в коде
  - *dogsled*: выявление чрезмернного количества игнорируемых возвращаемых значений
  - *bodyclose*: проверка закрытия тела ответа HTTP
  - *testpackage*: проверка, что тесты находятся в отдельном пакете от реализации
