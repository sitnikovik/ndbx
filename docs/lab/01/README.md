## Задание

Создайте простой HTTP-сервис с endpoint'ом для **healthcheck**.

### Требования к реализации

**REST API сервис:**

Реализуйте сервис с одним endpoint'ом `/health` для проверки доступности сервиса.

**Запрос:**

```http
GET http://localhost:8080/health
```

**Ожидаемый ответ:**

```json
{
    "status": "ok"
}
```

HTTP-статус: **200 OK**

### Конфигурация через переменные окружения

⚠️ Проект должен конфигурироваться через файл `.env.local`.

```sh
# Порт HTTP-сервера, на котором доступен сервис для запросов
APP_PORT=8080
```

### Запуск проекта

Проект должен запускаться одной командой:

```bash
make run
```

После успешного запуска сервис должен быть доступен:

```bash
curl http://localhost:8080/health
# Ожидается: {"status":"ok"}
```

И проект должен останавливаться командой:

```bash
make stop
```

## FAQ

**Q: Можно использовать что-то вместо Golang?**  
A: Да, любой язык. Главное — соблюсти
[требования по структуре проекта](https://github.com/sitnikovik/ndbx-template?tab=contributing-ov-file).

**Q: На каком порту должен слушать сервис?**  
A: Используйте порт из переменной `APP_PORT` в `.env.local` (по умолчанию 8080).

**Q: Можно ли выбрать другое название для endpoint'ов?
Например, вместо `/health` - `/healthcheck`?**  
A: Строго `/health` - автоматическая проверка ожидает именно этот путь.

**Q: Что делать, если проверка в Actions падает?**  
A: Смотрите логи: *Actions* → *ваш workflow* → *"Run healthcheck"* → *Show service logs*

**Q: Нужен ли Dockerfile?**  
A: Опционально, но рекомендуется. Можно использовать готовый образ в `docker-compose.yml`.
