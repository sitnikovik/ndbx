## Эндпоинты

### Рекомендации мероприятий

Добавьте новый эндпонит, который возвращает список рекомендованных мероприятий
на основе лайков пользователей

> 🔐 Только для авторизованных пользователей.
> Пользователь может запрашивать рекомендации только для себя.

```http
GET /users/{user_id}/recommendations HTTP/1.1
Host: localhost:8080
Content-Type: application/json
Content-Length: 999
```

**Ответ (ОК):**

```http
HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 999
```

> ⚠️ Рекомендации должны кэшироваться в Redis по принципу Cashe-aside (lazy),
чтобы не перестраивать подборку при каждом запросе

## Спецификация

### Схема данных в Neo4j

**Узлы:**

- `User` — пользователь системы
  - `id` *string* — его идентификатор в MongoDB
- `Event` — мероприятие
  - `id` *string* — его идентификатор в MongoDB
  - `title` *string* — название

**Связи:**

- `(u:User)-[:LIKED]->(e:Event)` — пользователь лайкнул мероприятие

- `LIKED` - связь, показывающая, что пользователь лайкнул то или иное событие

**Индексы:**

- `FOR (u:User) ON (u.id)` — для быстрого поиска пользователя
- `FOR (i:Interest) ON (i.name)` — для быстрого сопоставления интересов

**Алгоритм рекомендаций:**

Для пользователя А:

1. Что лайкнул А? → Мероприятие 1, Мероприятие 2
2. Кто ещё лайкнул Мероприятие 1? → Пользователь Б
3. Кто ещё лайкнул Мероприятие 2? → Пользователь Б, Пользователь В
4. Что лайкнул Пользователь Б? → Мероприятие 3, Мероприятие 4
5. Что лайкнул Пользователь В? → Мероприятие 5
6. Рекомендуем: Мероприятие 3, 4, 5 (которые А ещё не видел)

### Redis

**Ключ:**

```plaintext
recomms:{event_title_md5_hash}:recomms
```

`{event_title_md5_hash}` - хэш по алгоритму md5 для названия мероприятия.

Например, для "Алиса в стране чудес" ключ будет

```plaintext
event:ba80405c3ebccb9cb99791b47c2487f9:recomms
```

**Значение:**

## Конфигурация

```sh
NEO4J_HOST=neo4j
NEO4J_USER=neo4j
NEO4J_PASSWORD=password

# Время жизни рекомендаций в кэше
APP_RECOMMENDATIONS_TTL=60
```

## FAQ

**Q: Нужно ли учитывать рейтинг и отзывы в рекомендациях?**  
A: Нет, обойдемся лишь лайками (без дизлайков), так как любое добавление
может сильно поменять алгоритм рекомендаций

**Q: Сколько инстансов Neo4j поднимаем?**  
A: Один. Больше не требуется.

**Q: Нужно ли возвращать `count` в рекомендациях?**  
A: Нет, в рекомендациях только сами рекомендации.

**Q: При попытке получения рекомендаций другого пользователя возвращает 403?**  
A: Да. Хотя в других эндпоинтах при схожих действиях возвращаем 404,
в данном случае возвращаем именно 403
